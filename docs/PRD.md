## 1. 產品概述

### 1.1 產品簡介

Labor Law Assistant 是一個 **AI 驅動的台灣勞動法律知識庫與引導式問答系統**，旨在幫助一般勞工、HR 專員及中小企業主快速查詢、理解並應用台灣勞動法規。

### 1.2 問題陳述

| 問題 | 影響 |
|------|------|
| 勞動法規文字艱澀難懂 | 勞工無法理解自身權益 |
| 法規分散於多部法律 | 查詢困難、耗時 |
| 專業法律諮詢費用高 | 弱勢勞工無法負擔 |
| 資訊不對等 | 勞資爭議中勞方處於劣勢 |
| 數位落差 | 外籍勞工、年長者、身障者難以獲取資訊 |

### 1.3 解決方案

提供一個 **免費、無障礙、多語言** 的 AI 問答系統，讓所有人都能：
- 用白話文理解勞動法規
- 獲得可追溯來源的準確資訊
- 知道遇到問題時的下一步行動

### 1.4 產品定位

```
┌─────────────────────────────────────────────────────────────┐
│                     勞動法律資訊服務光譜                      │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  政府網站    →    本產品    →    法律諮詢    →    律師服務   │
│  (資訊陳列)      (AI 輔助)      (專業諮詢)      (代理訴訟)   │
│                                                             │
│  免費/被動       免費/主動      付費/專業       高價/專業     │
│  難以理解        白話互動       個案判斷       法律行動       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**本產品定位**：介於政府資訊與專業諮詢之間的「智慧橋樑」

### 1.5 核心價值主張

> **「讓每一位台灣勞工都能輕鬆理解自己的勞動權益，並知道如何保護自己。」**

## 2. 產品願景與目標

### 2.1 產品願景

成為台灣最值得信賴的勞動法律資訊平台，縮小勞資資訊落差，促進勞動權益保障。

### 2.2 產品使命

- **保護勞工優先**：系統立場明確站在保護勞工權益
- **資訊平等**：確保所有人（包括弱勢族群）都能獲取資訊
- **透明可信**：所有回答可追溯來源，清楚標示限制

### 2.3 商業目標

| 目標 | 指標 | 時程 |
|------|------|------|
| 建立使用者基礎 | MAU 10,000+ | 上線後 6 個月 |
| 達成產品市場契合 | NPS 30+ | 上線後 3 個月 |
| 成為信賴來源 | 回答滿意度 85%+ | 上線後 6 個月 |
| 涵蓋主要法規 | 8 大勞動法規 100% | MVP |

### 2.4 用戶權益憲章

本產品承諾遵守以下原則：

| 編號 | 承諾 |
|------|------|
| 1 | **保護勞工優先**：系統立場明確站在保護勞工權益，拒絕協助規避法規 |
| 2 | **隱私絕對保障**：不記錄個人辨識資訊，查詢紀錄絕不提供第三方 |
| 3 | **平等使用權**：確保所有人（包括身心障礙者、外籍勞工、年長者）都能使用 |
| 4 | **資訊透明**：明確標示資訊來源、更新日期、準確度 |
| 5 | **核心功能永久免費**：不向用戶收費 |
| 6 | **錯誤負責**：建立錯誤回報與修正機制，公開錯誤率 |
| 7 | **持續改進**：定期進行用戶研究，根據回饋優化系統 |
| 8 | **緊急支援**：提供真人協助轉介，不讓 AI 單獨處理危機情境 |
| 9 | **開放透明**：系統運作邏輯、演算法原則公開說明 |
| 10 | **社會責任**：主動推廣勞動權益教育，縮小資訊落差 |

| 用戶群體 | 潛在障礙 | 必要支援措施 | 優先級 |

| **年長者** | 字體小、操作複雜、不熟悉 AI | 大字體模式、簡化步驟 | P0 |
| **數位素養較低者** | 不理解系統運作 | 白話文說明、真人客服後援 | P0 |
| **低薪勞工** | 舊手機、網路流量有限 | 輕量化、離線快取、PWA | P1 |
| **夜班工作者** | 深夜使用、疲勞狀態 | 夜間模式、快速查詢捷徑 | P2 |

### 6.1 無障礙需求 (Accessibility)

| WCAG 2.1 原則 | 要求 | 驗收標準 |
|--------------|------|---------|
| **Perceivable** | 資訊可被多種感官接收 | - 所有圖片有替代文字<br>- 支援螢幕閱讀器<br>- 提供高對比模式<br>- 文字可縮放至 200% |
| **Operable** | 可用多種方式操作 | - 完整鍵盤導航<br>- 提供 Skip to content<br>- 無時間限制<br>- 支援語音輸入 |
| **Understandable** | 資訊清晰易懂 | - 語言簡單（國小六年級可理解）<br>- 一致的導航<br>- 錯誤訊息清楚 |
