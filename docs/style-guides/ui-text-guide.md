# UI 文字撰寫規範

## 目的

本指南規範系統介面所有文字的撰寫標準，確保一致性與使用者友善。

---

## 核心原則

### 1. 清楚 (Clear)
用戶一眼就能理解意思

### 2. 簡潔 (Concise)
用最少的字傳達最完整的意思

### 3. 友善 (Friendly)
專業但溫暖，不冷漠也不過度親暱

### 4. 一致 (Consistent)
相同功能使用相同文字

---

## 語氣與風格

### 整體語氣：專業但溫暖

```
我們的語氣：
✅ 友善的專業人士
✅ 可信賴的協助者
✅ 賦權而非命令

我們不是：
❌ 冷漠的機器人
❌ 過度熱情的推銷員
❌ 權威式的教條者
```

### 用詞選擇

| 情境 | ✅ 推薦 | ❌ 避免 |
|------|--------|--------|
| 稱呼用戶 | 您 | 你、使用者、閣下 |
| 系統自稱 | 系統、我們 | 本系統、敝司 |
| 請求動作 | 請、可以 | 必須、應該 |
| 錯誤訊息 | 出了點問題 | 錯誤、失敗 |
| 成功訊息 | 完成、已儲存 | OK、成功 |

## UI 元素文字規範

### 按鈕 (Buttons)

**原則**
- 動詞開頭
- 簡短有力（2-4 字為佳）
- 明確說明行為結果

**範例**

| 功能 | ✅ 推薦 | ❌ 避免 | 說明 |
|------|--------|--------|------|
| 提交 | 送出問題、查詢 | 確定、送出、Submit | 說明會發生什麼 |
| 取消 | 取消、返回 | 不要、Cancel | 簡潔明確 |
| 刪除 | 刪除 | 移除、刪掉 | 使用常見用語 |
| 確認 | 確定、我了解 | 好、OK | 清楚表達同意 |
| 展開 | 查看詳細、展開 | 更多、詳細內容 | 動詞開頭 |
| 分享 | 分享 | 分享此頁 | 簡潔 |

**特殊按鈕**

```
主要動作（Primary）：
✅ 「送出問題」
✅ 「開始查詢」
✅ 「立即計算」

次要動作（Secondary）：
✅ 「取消」
✅ 「返回」
✅ 「稍後再說」

危險動作（Destructive）：
✅ 「確定刪除」
✅ 「清除全部」
```

### 標題與說明 (Headings & Descriptions)

**頁面標題**
- 簡短明確（4-8 字）
- 描述頁面功能
- 使用名詞或動詞片語

```
✅ 好標題：
「查詢勞動法規」
「加班費計算器」
「我的查詢記錄」

❌ 太模糊：
「查詢」
「計算」
「記錄」
```

**說明文字**
- 補充標題資訊
- 一句話說明用途
- 不超過 20 字

```
✅ 好說明：
「輸入您的問題，AI 會幫您找到相關法規」
「根據您的薪資和工時，自動計算加班費」

❌ 太冗長：
「這是一個可以幫助您查詢台灣勞動相關法律規定的智慧問答系統，
您可以在下方輸入框輸入您想要詢問的問題...」
```

### 表單標籤 (Form Labels)

**原則**
- 簡短清楚
- 避免冒號（行動裝置上佔空間）
- 必要時加上輔助說明

**範例**

```html
✅ 推薦：
<label>月薪（新台幣）</label>
<input type="number" placeholder="例如：30000">
<span class="hint">請輸入稅前月薪</span>

❌ 避免：
<label>請輸入您的月薪資：</label>
<input type="number">
```

**常用標籤**

| 欄位類型 | 標籤文字 | 輔助說明（如需要） | Placeholder |
|---------|---------|-----------------|-------------|
| 薪資 | 月薪（新台幣） | 請輸入稅前月薪 | 例如：30000 |
| 工作時間 | 每日工時 | 正常工作時間 | 例如：8 |
| 日期 | 到職日期 | | 選擇日期 |
| 職稱 | 職位 | | 例如：工程師 |
| 問題 | 您的問題 | | 例如：加班費怎麼算？ |

---

### Placeholder 文字

**原則**
- 提供範例而非指示
- 使用「例如：」開頭
- 不重複 label 內容

```
✅ 好 Placeholder：
「例如：加班費怎麼算？」
「例如：30000」
「例如：工程師」

❌ 不好的 Placeholder：
「請在此輸入您的問題」
「月薪」
「輸入職位」
```

### 錯誤訊息 (Error Messages)

**原則**
1. 說明發生了什麼
2. 解釋為什麼
3. 告訴如何修正

**結構**
```
[問題] - [原因] - [解決方法]
```

**範例**

| 情境 | ✅ 好的錯誤訊息 | ❌ 不好的錯誤訊息 |
|------|---------------|-----------------|
| 必填欄位空白 | 「請輸入月薪才能計算加班費」 | 「錯誤：欄位不得空白」 |
| 格式錯誤 | 「月薪請輸入數字，例如：30000」 | 「格式錯誤」 |
| 數值超出範圍 | 「月薪請輸入 1-1000000 之間的數字」 | 「數值錯誤」 |
| 網路錯誤 | 「網路連線中斷，請檢查您的網路後重試」 | 「網路錯誤」 |
| 系統錯誤 | 「系統暫時無法處理，請稍後再試或聯繫客服」 | 「Error 500」 |

**錯誤訊息範本**

```markdown
# 輕微錯誤（用戶可自行修正）
⚠️ [簡短描述]
[具體說明]
[解決方法]

範例：
⚠️ 請輸入月薪
計算加班費需要您的月薪資訊。
請在上方欄位輸入您的月薪（例如：30000）

# 嚴重錯誤（需要協助）
❌ [發生什麼事]
[可能原因]
您可以：
1. [解決方法 1]
2. [解決方法 2]
3. [聯繫客服]

範例：
❌ 查詢失敗
系統暫時無法處理您的問題。
您可以：
1. 稍後再試
2. 重新整理頁面
3. 聯繫客服：service@example.com
```

### 成功訊息 (Success Messages)

**原則**
- 簡短肯定
- 說明完成了什麼
- 建議下一步（如需要）

**範例**

```
✅ 已儲存您的查詢
✅ 計算完成
✅ 已送出您的回饋，感謝您！

加上下一步：
✅ 已儲存您的查詢
   您可以在「查詢記錄」中查看。

✅ 計算完成
   您可以列印或分享此結果。
```

### 提示訊息 (Hints & Tips)

**原則**
- 協助用戶理解或決策
- 簡短友善
- 不過度打擾

**範例**

```
💡 小提醒：
月薪需要除以 240（小時）才是您的時薪。

💡 不確定怎麼問？
試試選擇下方的問題範本。

💡 第一次使用？
查看快速入門指南（2 分鐘）
```

---

### 空狀態 (Empty States)

**原則**
- 說明為什麼是空的
- 提供明確的行動呼籲

**範例**

```
📋 還沒有查詢記錄

開始查詢勞動法規，
您的記錄會自動儲存在這裡。

[開始查詢]

---

🔍 找不到相關結果

試試：
• 使用不同的關鍵字
• 選擇問題範本
• 查看常見問題

[查看常見問題]
```

---

### 載入狀態 (Loading States)

**原則**
- 告知進度
- 管理期待
- 保持友善

**範例**

```
⏳ 正在查詢法規...
⏳ 計算中...
⏳ AI 正在思考...（預計 5 秒）
⏳ 正在產生回答...
```

**長時間載入**
```
⏳ 正在分析您的問題...

這可能需要 10-15 秒，
感謝您耐心等候。

💡 小提醒：複雜問題需要更多時間處理
```

## 多語言考量

### 繁體中文規範

**數字**
- ✅ 使用阿拉伯數字：8 小時
- ❌ 避免中文數字：八小時（除非法條原文）

**單位**
- ✅ 中文單位：元、小時、天
- ❌ 英文單位：NTD、hrs、days

**標點符號**
- ✅ 中文標點：，。！？
- ❌ 混用：，. !?

**括號**
- ✅ 全形括號：（說明）
- ❌ 半形括號：(說明)

**引號**
- ✅ 中文引號：「引用」
- ❌ 英文引號："引用"

### 簡易中文版本

針對語言能力較低者，應：
- 使用更簡單的詞彙
- 句子更短（< 15 字）
- 避免法律術語
- 多用視覺輔助

**範例對照**

| 標準版 | 簡易版 |
|-------|-------|
| 延長工作時間 | 加班 |
| 僱主應給付工資 | 老闆要給薪水 |
| 您可以向勞工局申訴 | 可以打電話給勞工局 |

## 無障礙考量

### 螢幕閱讀器

**圖示必須有 alt text**
```html
✅ 推薦：
<button>
  <icon alt="警告" />
  緊急協助
</button>

❌ 避免：
<button>
  <icon />
  <!-- 螢幕閱讀器讀不到圖示意義 -->
</button>
```

**連結必須有意義**
```html
✅ 推薦：
<a href="#">查看完整隱私政策</a>

❌ 避免：
<a href="#">點這裡</a>查看隱私政策
```

## 文字長度限制

| UI 元素 | 建議長度 | 最大長度 | 超過時處理 |
|---------|---------|---------|-----------|
| 按鈕 | 2-4 字 | 8 字 | 改用圖示+文字 |
| 標題 | 4-8 字 | 15 字 | 使用副標題 |
| 說明 | 10-20 字 | 30 字 | 分段或使用提示 |
| 錯誤訊息 | 15-30 字 | 50 字 | 分段說明 |
| Toast 通知 | 10-15 字 | 25 字 | 使用通知中心 |

## 檢查清單

發布前檢查：

### 清楚度
- [ ] 用戶能立即理解意思
- [ ] 無法律術語（或有解釋）
- [ ] 無技術術語
- [ ] 無縮寫（或有解釋）

### 簡潔度
- [ ] 無贅字
- [ ] 符合長度限制
- [ ] 一句話說明一件事

### 友善度
- [ ] 語氣溫暖
- [ ] 避免命令式
- [ ] 有同理心

### 一致性
- [ ] 與其他頁面用詞一致
- [ ] 按鈕文字符合規範
- [ ] 語氣一致

### 無障礙
- [ ] 圖示有 alt text
- [ ] 連結有意義的文字
- [ ] 顏色不是唯一區別

### 多語言
- [ ] 中文標點正確
- [ ] 數字格式正確
- [ ] 適合翻譯（如需要）
