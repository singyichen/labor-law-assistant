## User Advocacy Framework

### Understand
- Who are the users?
- What are their goals?
- What context do they operate in?
- What are their pain points?

### Evaluate
- Does this solve a real user problem?
- Is it easy to understand and use?
- Does it meet accessibility standards?
- Does it align with user expectations?

### Advocate
- Present user perspective clearly
- Back arguments with data
- Propose user-friendly alternatives
- Balance business and user needs

### Validate
- Test with real users
- Measure user satisfaction
- Iterate based on feedback

## Usability Heuristics Checklist

| Heuristic | Question |
|-----------|----------|
| Visibility | Can users see system status? |
| Match | Does it use familiar language? |
| Control | Can users undo and redo? |
| Consistency | Is it consistent throughout? |
| Error Prevention | Does it prevent errors? |
| Recognition | Is it easy to recognize vs recall? |
| Flexibility | Does it support different skill levels? |
| Aesthetics | Is the design clean and minimal? |
| Error Recovery | Can users recover from errors easily? |
| Help | Is help available when needed? |

## Review Checklist

- User goals clearly understood
- Pain points identified
- Usability issues documented
- Accessibility evaluated
- User feedback incorporated
- Recommendations are actionable
- Trade-offs are balanced
- Metrics defined for success
